# Альтернативные картографические API

## Текущая ситуация

Приложение использует Google Maps API для:
- Отображения карты с маркерами
- Рисования полигона для поиска
- Построения маршрута (линии между точками)
- Геокодирования адресов

## Проблема

Для построения маршрутов общественного транспорта в Москве лучше подходят российские сервисы:
- Яндекс.Карты
- 2ГИС

## Варианты решения

### Вариант 1: URL-ссылки (реализован)

Самый простой подход - добавить кнопки для открытия маршрута во внешних приложениях.

**Яндекс.Карты (рабочий формат):**
```
https://yandex.ru/maps/213/moscow/?mode=routes&rtext=lat1,lng1~lat2,lng2~lat3,lng3&rtt=auto
```
Параметры:
- `rtext` - координаты точек через `~`
- `mode=routes` - режим маршрута
- `rtt=auto` - тип маршрута: auto (общественный + авто), pd (пешком), masstrans (только транспорт)
- Можно добавить `&ll=lon,lat` для начального центра карты
- `&z=10` - зум

**2ГИС (базовый формат):**
```
https://2gis.ru/route/from/lat1,lng1/to/lat2,lng2;lat3,lng3
```

### Вариант 1б: 2GIS API для маршрутов

2ГИС предоставляет API для построения маршрутов:
- Бесплатный API для маршрутизации
- Документация: https://api.2gis.ru/doc/
- Позволяет встраивать маршруты без перехода на внешний сайт

**Преимущества 2GIS API:**
- Бесплатное использование
- Хорошая маршрутизация общественного транспорта
- Можно получить данные маршрута (время в пути, расстояние)

**Минусы:**
- Требует регистрации и получения API ключа
- Ограничения по запросам

**Пример использования API:**
```
https://api.2gis.ru/2.0/router/route?points=lat1,lng1;lat2,lng2&type=pedestrian
```

**Плюсы:**
- Бесплатно
- Не требует API ключей
- Простая реализация
- Использует нативные приложения с лучшими данными по транспорту

**Минусы:**
- Пользователь уходит из приложения
- Нужно переключаться между вкладками

### Вариант 2: Яндекс Maps JavaScript API

**Возможности:**
- ✅ Рисование полигонов (GeoObjects - Polygon)
- ✅ Маркеры (Placemark)
- ✅ Линии пути (Polyline)
- ✅ Маршрутизация общественного транспорта (MultiRoute)
- ✅ Геокодирование
- ✅ Поиск по карте

**Стоимость:**
- Бесплатно до 1000 запросов в сутки
- Далее ~195₽ за 1000 запросов

**Плюсы:**
- Полная замена Google Maps
- Лучшие данные по Москве
- Маршрутизация транспорта
- Русскоязычная поддержка

**Минусы:**
- Требует переписывания кода
- Платное использование при большом трафике
- Нужен API ключ

### Вариант 2б: Полная интеграция маршрутов через Яндекс JS API

Для реализации построения маршрута общественного транспорта с получением геометрии:

**Подключение API:**
```html
<script src="https://api-maps.yandex.ru/2.1/?apikey=YOUR_API_KEY&lang=ru_RU"></script>
```

**Создание карты:**
```typescript
ymaps.ready(init);

function init() {
    const map = new ymaps.Map('map', {
        center: [55.7558, 37.6173],
        zoom: 10
    });
}
```

**Построение маршрута общественного транспорта:**

```typescript
// Массив точек маршрута
const wayPoints = [
    [55.782185, 37.483117],  // lat, lng
    [55.792244, 37.598474],
    [55.780637, 37.671258],
    [55.810808, 37.731683]
];

// Создание мультимаршрута
const multiRoute = new ymaps.multiRoute.MultiRoute({
    referencePoints: wayPoints,
    params: {
        // Тип маршрута: masstransit - общественный транспорт
        routingMode: 'masstransit',
        // Используем общественный транспорт
        transportMode: 'publicTransport',
        // Включить расчет времени в пути
        via: wayPoints.slice(1, -1)
    }
});

// Добавление маршрута на карту
map.geoObjects.add(multiRoute);

// Подписка на событие готовности маршрута
multiRoute.events.add('ready', function() {
    // Получение сегментов маршрута
    const routes = multiRoute.getRoutes();
    
    // Перебор сегментов (альтернативные маршруты)
    routes.each(function(route) {
        const segments = route.getSegments();
        
        // Перебор сегментов маршрута
        segments.each(function(segment) {
            // Получение геометрии сегмента (координаты)
            const geometry = segment.getPaths();
            const coordinates = geometry.getCoordinates();
            
            // Тип транспорта (метро, автобус, etc)
            const transportType = segment.getType();
            
            // Время в пути
            const duration = segment.getDuration();
        });
    });
});
```

**Получение геометрии маршрута:**

```typescript
// После загрузки маршрута
multiRoute.events.add('ready', function() {
    // Получаем все маршруты (обычно 1-3 варианта)
    const routes = multiRoute.getRoutes();
    
    routes.each(function(route) {
        // Получаем сегменты
        const segments = route.getSegments();
        
        // Собираем все координаты в один массив
        const allCoordinates: number[][] = [];
        
        segments.each(function(segment) {
            // Метод getPath() возвращает путь (Path) с координатами
            const path = segment.getPath();
            const coords = path.getCoordinates();
            allCoordinates.push(...coords);
        });
        
        console.log('Маршрут:', allCoordinates);
    });
});
```

**Отрисовка маршрута на карте:**

Можно использовать полученные координаты для отрисовки собственной линии:

```typescript
// Создание линии по координатам маршрута
const routeLine = new ymaps.Polyline(allCoordinates, {
    strokeColor: '#4a90d9',
    strokeWidth: 4,
    strokeOpacity: 0.8
});

map.geoObjects.add(routeLine);
```

**Добавление маркеров точек:**

```typescript
// Для каждой точки маршрута
wayPoints.forEach((coords, index) => {
    const placemark = new ymaps.Placemark(coords, {
        iconContent: String(index + 1),
        balloonContent: `Точка ${index + 1}`
    }, {
        preset: 'islands#circleDotIcon',
        iconColor: '#4a90d9'
    });
    
    map.geoObjects.add(placemark);
});
```

**Интеграция с текущим приложением:**

План перехода:
1. Добавить условие выбора карты (Google vs Яндекс)
2. Создать абстракцию MapProvider с интерфейсом:
   - `showPolygon()` / `hidePolygon()`
   - `addMarker()` / `removeMarker()`
   - `drawRoute()` / `clearRoute()`
   - `getCoordinates()`
3. Реализовать GoogleMapProvider и YandexMapProvider
4. Использовать инъекцию зависимостей для выбора провайдера

**Преимущества полной интеграции:**
- Маршрут строится прямо в приложении
- Можно показать остановки общественного транспорта
- Лучший UI для Москвы

**Сложности:**
- Переписывание логики карты (~500 строк)
- Нужен API ключ Яндекса
- Тестирование

### Вариант 3: 2GIS API

**Возможности:**
- ✅ Слой общественного транспорта
- ✅ Маршрутизация
- ✅ Геокодирование

**Стоимость:**
- Бесплатно для многих случаев использования

**Плюсы:**
- Отличное покрытие России
- Хорошая маршрутизация транспорта
- Бесплатный тариф

**Минусы:**
- Менее распространённый API
- Меньше English документации
- Требует переписывания кода

---

### Вариант 4: OpenStreetMap (бесплатно)

**Инструменты:**
- Leaflet.js - для отображения карты
- Overpass API - для получения данных (остановки)
- OSRM или GraphHopper - для маршрутизации

**Плюсы:**
- Полностью бесплатно
- Не требует API ключа
- Open source

**Минусы:**
- Маршрутизация общественного транспорта ограничена
- Данные могут быть менее точными для России
- Требует больше кода

## Рекомендации

### Краткосрочно (реализовано):
Использовать URL-ссылки для быстрого доступа к маршрутам в Яндекс.Картах и 2ГИС.

### Среднесрочно:
Рассмотреть переход на Яндекс Maps JavaScript API если:
- Трафик превысит бесплатный лимит Google Maps
- Потребуется более глубокая интеграция
- Нужны будут остановки транспорта на карте

### Долгосрочно:
- Сделать абстракцию над картой (pattern Strategy)
- Позволить переключаться между провайдерами
- Можно использовать OpenStreetMap как бесплатный backup
